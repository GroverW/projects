What is a stateless component?

A stateless component is one that does not depend on any other "state" to produce a result, like the functional programming paradigm.

What is the difference between the equality operators == and ===?

== will try to coerce the right side of the expression into the same type as the left side, so 1 == '1' will be true, while === is strict equals and returns false. == is also what people seem to mostly make jokes about JavaScript for.

What is the difference between the postfix i++ and prefix ++i increment operators?

Order of operations. i++ happens after the expression it's apart of is executed, while ++i is the same as saying i += 1. And Douglas Crockford says don't use either.

Create a function batches that returns the maximum number of whole batches that can be cooked from a recipe.

Ok so basically return either the smallest Math.floor(availableIngredient / recipeIngredient) or 0 if you don't have one of them. 


  const batches = (recipe, available) => {
    let maxBatches = Infinity;

    for(let ingredient in recipe) {
      if(!available[ingredient]) return 0;

      const numBatches = Math.floor(available[ingredient] / recipe[ingredient]);
      maxBatches = Math.min(maxBatches, numBatches);
    }

    return maxBatches;
  }

What is the difference between synchronous and asynchronous code in JavaScript?

Synchronous code gets executed in a blocking manner off the call-stack. So when the interpreter reaches a line of code (like a function), it places it on the call-stack, executes until it reaches another function or callback or whatever, pushes that one to the call-stack, and continues until it reaches the end of the function on top of the stack, which gets popped off and executed until the stack is empty.

Asynchronous code starts on the call-stack, but gets immediately popped off and moved to the event loop, where any asynchronous calls (like api calls) are handled by web workers. Once complete, it gets moved to the event queue which waits until the call-stack is empty before popping functions from the queue back to the call-stack for execution.

How do you compare two objects in JavaScript?

Three ways I can think of - use === to see if it's literally the same object, use JSON.stringify to see if they have the same structure, or iterate through recursively to check each value against one another.
